# 跟着 Claude 学英语 — 提示词分析报告

> 基于 Claude Code CLI 源码 (`claude-code-analyze/src/`) 中所有用户可见英文文本的深度分析
>
> 分析日期：2026-04-05

---

## 一、项目概述

Claude Code 是 Anthropic 的终端 AI 编程助手，使用 TypeScript + React/Ink 构建终端 UI。源码中包含 **350+** 条用户可见的英文文本，涵盖状态提示、权限确认、错误处理、欢迎引导、命令描述等场景。

本报告将这些英文文本分为 **9 大主题**，每张图对应一个主题，适合制作"跟着 Claude 学英语"系列卡片。

---

### 主题 2：Permission Prompts（权限确认）— "Do you want to..."

**来源文件**: `src/components/permissions/`

这是人机交互中最核心的对话模式。

| 英文原文 | 中文 | 句型分析 |
|----------|------|----------|
| Do you want to proceed? | 你要继续吗？ | 最经典的确认句 |
| Do you want to make this edit to [file]? | 你要对这个文件做修改吗？ | Do you want to + 动词原形 |
| Enter plan mode? | 进入计划模式？ | 动词原形 + 问号（简洁版） |
| Exit plan mode? | 退出计划模式？ | 同上 |
| Ready to code? | 准备好写代码了？ | 形容词 + 问号 |
| Yes, enter plan mode | 好的，进入计划模式 | 确认选项 |
| No, start implementing now | 不，现在就开始实现 | 拒绝选项 |
| Yes, and use auto mode | 是，并使用自动模式 | Yes, and... 递进句式 |
| Yes, and bypass permissions | 是，并绕过权限 | Yes, and... 递进句式 |
| Yes, auto-accept edits | 是，自动接受编辑 | 简洁确认 |
| Yes, manually approve edits | 是，手动批准编辑 | 简洁确认 |
| No, keep planning | 不，继续计划 | No, + 动词 |
| No, refine with Ultraplan on Claude Code on the web | 不，在网页版用 Ultraplan 精修 | 长路径指引 |
| Claude Code needs your approval for the plan | Claude Code 需要你批准这个计划 | 主语 + needs + your + 名词 |
| Claude Code wants to enter plan mode | Claude Code 想进入计划模式 | 主语 + wants to + 动词 |
| Claude needs your permission to use {toolName} | Claude 需要你允许使用 {toolName} | needs your permission to + 动词 |
| Claude Code needs your attention | Claude Code 需要你的关注 | needs your attention |
| Claude Code won't ask before using allowed tools. | Claude Code 使用已允许的工具前不会询问。 | won't + 动词 |
| Claude Code will always ask for confirmation before using these tools. | Claude Code 使用这些工具前总会要求确认。 | will always + 动词 |
| Claude Code will always reject requests to use denied tools. | Claude Code 总会拒绝使用被禁止的工具。 | will always + 动词 |
| Yes, and don't ask again for [duration] | 好的，这段时间内别再问了 | Yes, and don't... |
| Allow network connection to {host}? | 允许连接到 {host}？ | Allow + 名词 + to... |
| Network request outside of sandbox | 沙箱外的网络请求 | 名词短语（标题） |
| Dangerous {command} operation detected... | 检测到危险的 {command} 操作 | 过去分词 + 名词 |
| Permission denied by hook | 权限被 hook 拒绝 | 被动语态 |

**学习要点**:
- `Do you want to...` 是最标准的确认句型
- `Yes, and...` 用于递进确认（同意 + 额外操作）
- Claude Code 把自己称为第三人称 "Claude Code"，非常礼貌
- `won't ask before using` 和 `will always ask before using` 形成对比

- Claude Code 几乎从不说 "Sorry" — 用 `Please` + 解决方案代替

## 三、语言设计总结

### Claude Code 的英文语言风格特征

1. **动词优先**: 命令描述统一用动词原形开头
2. **中点分隔**: 用 `·` 分隔错误描述和解决方案
3. **Please + 具体操作**: 礼貌且有用，不只是 "Please try again"
4. **委婉时态**: 用 `is experiencing` 代替 `is overloaded`，用 `may be` 代替 `is`
5. **随机人情味**: 随机告别语、200 个加载动词、幽默的 spinner
6. **拟人化**: Claude 称自己为 "Claude"，用 "Claude will think"
7. **口语化按钮**: "Got it, thanks!" 比 "OK" 更有人情味
8. **祈使句引导**: "Use /clear to..." 而不是 "You can use /clear to..."
9. **条件句 + 建议**: "If you are behind a proxy, set NODE_EXTRA_CA_CERTS"
10. **永不道歉**: 用 `Please` + 解决方案代替 "Sorry"
