# 技巧八：会话记忆系统

> 核心思想：压缩之后还能记得什么？用结构化模板确保关键信息不丢

---

## 架构

`src/services/SessionMemory/prompts.ts` 实现了跨压缩的会话记忆持久化。

```
压缩前：
  [长对话...] → [压缩摘要]

压缩后 + 记忆：
  [压缩摘要] + [Session Memory 文件]
```

Session Memory 是一个**独立文件**，不是压缩摘要的一部分。它在压缩后仍然存在，提供额外的持久化信息。

## 十段式记忆模板

```markdown
# {{title}}

## Current State
当前状态概述...

## Task
任务规格说明...

## Files / Functions
涉及的文件和函数...

## Workflow
工作流程...

## Errors / Corrections
错误和修正记录...

## Codebase Documentation
代码库文档引用...

## Learnings
学习到的经验...

## Key Results
关键成果...

## Worklog
工作日志...
```

### 预算控制

```typescript
max_tokens_per_section = 2000
max_total_tokens = 12000
```

每个 section 最多 2000 tokens，总计最多 12000 tokens。

**Why**: 记忆文件应该轻量，不应该和原始对话一样长。

---

## 更新机制

更新 Session Memory 时，模型只能使用 **Edit 工具**：

```
"You may ONLY use the Edit tool. No other tools are available."
```

**Why**:
1. 防止模型意外覆盖整个文件（Write 工具会替换全部内容）
2. 鼓励增量更新（只修改变化的部分）
3. 降低 token 消耗（Edit 比 Write 生成更少的输出）

### 模板结构保护

```
"Never modify headers or italic descriptions — only update the content."
```

标题和斜体描述是模板结构，不能改。只能改下面的内容。

**Why**: 如果模型改了标题，后续更新时可能找不到对应 section。

## 提示词工程技巧总结

| 技巧 | 具体做法 | 效果 |
|------|---------|------|
| 结构化模板 | 10 个固定 section | 保证记忆完整性 |
| Token 预算 | 每 section ≤ 2000，总计 ≤ 12000 | 防止记忆膨胀 |
| Edit-only 限制 | 只能用 Edit 工具更新 | 防止覆盖，鼓励增量更新 |
| 结构锁定 | 标题不可修改 | 保证后续更新能找到对应位置 |
| 分离存储 | 独立文件，不在压缩摘要内 | 压缩不影响记忆 |

---

## 与 Compact 的区别

| 维度 | Compact（压缩） | Session Memory（记忆） |
|------|----------------|----------------------|
| 目的 | 节省上下文窗口 | 跨压缩保留状态 |
| 格式 | 9 段式摘要 | 10 段式模板 |
| 生命周期 | 每次压缩重新生成 | 跨压缩持久化 |
| 存储位置 | 替换原对话 | 独立文件 |
| Token 上限 | 无硬性限制 | 12000 tokens |
