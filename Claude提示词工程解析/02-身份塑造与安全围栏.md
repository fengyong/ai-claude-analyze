# 技巧二：身份塑造 & 安全围栏

> 核心思想：先告诉 AI "你是谁"，再告诉 AI "你不能做什么"

---

## 身份定义（你是谁）

`src/constants/system.ts` 定义了三种身份前缀：

| 前缀 | 场景 |
|------|------|
| `DEFAULT_PREFIX` | 标准 Claude Code CLI |
| `AGENT_SDK_CLAUDE_CODE_PRESET_PREFIX` | Agent SDK + Claude Code 预设 |
| `AGENT_SDK_PREFIX` | 纯 Agent SDK 调用 |

每个前缀包含：
- **身份声明**："You are Claude, an AI assistant made by Anthropic"
- **产品定位**："official CLI for Claude"
- **能力范围**：软件工程任务
- **归因头**：`cc_version` + `cc_entrypoint`，用于 API 请求追踪

归因头由 `getAttributionHeader()` 生成，格式为：
```
You are powered by the model {model_name}.
The most recent Claude model family is Claude 4.5/4.6.
```

## 安全围栏（你不能做什么）

`src/constants/cyberRiskInstruction.ts` 定义了一段精炼的安全边界指令：

### 允许的安全活动

- 授权的安全测试（渗透测试、红队演练）
- 防御性安全研究
- CTF 比赛
- 教育场景

### 明确拒绝

- 破坏性技术
- DoS 攻击
- 大规模目标攻击
- 供应链攻击
- 检测规避（用于恶意目的）

### 双重用途工具的处理

对于 C2 框架、凭证测试、漏洞利用开发等双重用途工具，要求提供**明确的授权上下文**（如渗透测试合同编号、CTF 比赛名称）。

---

## 网络安全指令的嵌入位置

安全指令被嵌入在系统 prompt 的**身份层**（`getSimpleIntroSection()`），这意味着：

1. 它在**最前面**，是模型最先"看到"的约束
2. 它属于**静态层**，可缓存
3. 它对所有请求生效，不可被后续指令覆盖

```
System Prompt 结构：

[身份声明]
[安全指令]        ← 在这里，位置靠前 = 优先级高
[行为准则]
[编码风格]
...
```

---

## 提示词工程技巧总结

| 技巧 | 具体做法 | 效果 |
|------|---------|------|
| 身份前置 | 安全指令紧跟身份声明之后 | 模型最先接收约束，权重最高 |
| 明确列举 | "你可以做 X, Y, Z" + "你不能做 A, B, C" | 比模糊描述更有效 |
| 授权上下文要求 | 双重用途工具需要明确授权 | 不是"一刀切"禁止，而是要求证明 |
| 位置决定权重 | 放在 prompt 前段（静态层） | 系统级约束，不可被用户消息覆盖 |
